# Дайджест Piterstory

Веб-страница дайджеста новостей недели в фирменном стиле Piterstory.

## 🚀 Как опубликовать на GitHub Pages (5 минут)

### Шаг 1: Регистрация на GitHub
1. Переходи на [github.com](https://github.com)
2. Нажми **Sign up** (если ещё нет аккаунта)
3. Укажи email, придумай пароль, выбери username

### Шаг 2: Создание репозитория
1. Нажми зелёную кнопку **New** (или перейди на https://github.com/new)
2. Заполни:
   - **Repository name**: `piterstory-digest` (или любое другое имя)
   - Поставь галочку **Public**
   - Поставь галочку **Add a README file**
3. Нажми **Create repository**

### Шаг 3: Загрузка файлов
1. В созданном репозитории нажми **Add file** → **Upload files**
2. Перетащи ВСЕ файлы из папки `piterstory-digest`:
   - `index.html`
   - Папку `images` со всеми картинками
3. Внизу страницы нажми **Commit changes**

### Шаг 4: Включение GitHub Pages
1. В репозитории перейди в **Settings** (вкладка справа вверху)
2. В левом меню найди **Pages**
3. В разделе **Source** выбери:
   - **Branch**: `main`
   - **Folder**: `/ (root)`
4. Нажми **Save**

### Шаг 5: Получение ссылки
Через 1-2 минуты страница будет доступна по адресу:
```
https://ВАШ_USERNAME.github.io/piterstory-digest/
```

Например, если твой username `piterstory`, то ссылка будет:
```
https://piterstory.github.io/piterstory-digest/
```

## ❓ Если что-то не работает

**Картинки не отображаются?**
- Проверь, что загрузил папку `images` вместе с `index.html`
- Структура должна быть: `index.html` в корне, `images` рядом с ним

**Страница не открывается?**
- Подожди 2-3 минуты после включения GitHub Pages
- Проверь правильность ссылки: `username.github.io/название-репозитория`

**Хочу изменить текст?**
- Открой `index.html` в любом текстовом редакторе
- Измени нужный текст
- Загрузи обновлённый файл обратно в GitHub
